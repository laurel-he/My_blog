新建feature/hotfix进行项目开发

  1. 更新本地代码仓库缓存
    git remote update

  2. 切换到本地develop分支
    git checkout develop

  3. 更新本地develop分支到最新
    git merge origin/develop
    (采用fork仓库开发的, 使用 git merge upstream/develop 进行合并)

  4. 使用当前年月日格式创建对应的feature/hotfix
    feature/20171012_xxxx
    hotfix/20171012_xxxxx2
    例: 创建一个2017年10月12日的酒店优化分支
    git branch feature/20171012_hotel_enhance

  5. 开发进行中和开发完成后, 都需要进行 1 操作, 然后把 develop的更新合并进来, 保持代码更新,
    同时, 需要在此处解决合并冲突.
    git checkout feature/20171012_hotel_enhance
    git remote update
    git merge origin/develop (采用fork 仓库开发的, 使用git merge upstream/develop)

  6. 开发完成后, 推送当前开发分支至远端服务器, 同时在github, git.tff.bz 创建Pull Request(PR)到develop 分支.
    在发送提测邮件时, 带上该项目的 PR 链接地址. 该操作是为了方便后续项目代码合并review.
    git push origin feature/20171012_hotel_enhance
    (采用fork 仓库开发的, 使用 git push upstream feature/20171012_hotel_enhance)

  7. 测试开始测试之前, 会邮件通知开始测试该feature/hotfix, 需要开发再次进行 5,6 操作, 把代码更新
    git checkout feature/20171012_hotel_enhance
    git remote update
    git merge origin/develop (采用fork 仓库开发的, 使用git merge upstream/develop, 后面不再复述fork的操作流程)
    git push origin feature/20171012_hotel_enhance

  8. 测试通过后, 邮件通知开发进行打 release操作. 需要再次更新代码到最新
    git checkout feature/20171012_hotel_enhance
    git remote update
    git merge upstream/develop
    git push origin feature/20171012_hotel_enhance
    git branch release/20171026_hotel_enhance
    git push origin release/20171026_hotel_enhance

  9. 测试开始测试release, 如果有问题, 开发需要在该release上进行修改, 并且每次需要重复合并develop代码来保持代码更新.

  10. 测试无误后, 测试将发出测试通过, 准备上线邮件, 研发需要和测试沟通是否有最新项目已经提前上线, 来决定是否需要再次更新代码.
  11. 上线人员检查 PR 的链接, 对相关项目代码进行 review, 确认无误后, 进行上线. 执行完提测邮件里面提到的 sql, cem变量, 脚本修复, 代码部署等操作后, 发送上线完成邮件通知.
  12. 测试人员回归测试, 验收完成后, 发送线上验证通过通知.
  13. 开发收到验证完成通知后, 把 release分支回合到 feature 分支, 然后要求项目负责人进行合并到
    develop/master操作.
  14. 项目负责人在github, git.tff.bz 的PR 上review 代码. 确认没问题后, 合并进入 develop 分支, 同时创建 develop到 master的PR, 进行再次合并.

##补充
1 删除分支
git branch –delete –remotes origin/feature/20190214_monthly
git push origin :feature/20190214_monthly
2 add之后回退
git reset HEAD
3 撤销commit
git commit –amend
4 撤销已经commit但是没有push
git log 找到commit id
git reset –hard commit_id 完成撤销,同时将代码恢复到前一commit_id 对应的版本
git reset commit_id 完成Commit命令的撤销，但是不对代码修改进行撤销，可以直接通过git commit 重新提交对本地代码的修改
